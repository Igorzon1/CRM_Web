# Histórias de Usuário (User Stories)

Este documento traduz os requisitos funcionais em um formato Ágil (User Stories), focado no valor que a funcionalidade traz para a persona. Isso guiará as entregas (Sprints/Kanban).

## Épico 1: Autenticação e Acesso
- **US01:** Como **vendedor**, eu quero **fazer login com meu email e senha** para que **eu possa acessar meus clientes de forma segura e privada**.
- **US02:** Como **vendedor logado**, eu quero **poder fazer logout** para que **minha conta não seja acessada indevidamente no meu computador**.

## Épico 2: Gestão de Base de Clientes
- **US03:** Como **vendedor**, eu quero **cadastrar um novo cliente informando Nome, Email, Telefone e Documento** para que **eu possa iniciar o processo de prospecção**.
- **US04:** Como **vendedor**, eu quero **ver a lista de todos os meus clientes cadastrados, com pesquisa por nome** para que **eu encontre facilmente com quem preciso falar hoje**.
- **US05:** Como **vendedor**, eu quero **adicionar notas de texto ao perfil do cliente** para que **eu registre o que foi falado na última ligação**.

## Épico 3: Funil de Vendas (Pipeline)
- **US06:** Como **vendedor**, eu quero **visualizar minhas oportunidades de negócio em um formato visual (Board/Kanban)** para que **eu saiba rapidamente quantas negociações estão em andamento**.
- **US07:** Como **vendedor**, eu quero **mover uma oportunidade da coluna "Prospecção" para "Proposta Enviada"** para que **meu funil reflita a realidade atual da venda**.
- **US08:** Como **gerente de vendas**, eu quero **marcar uma oportunidade como "Ganha" e informar o valor fechado** para que **o sistema contabilize o atingimento das minhas metas**.
